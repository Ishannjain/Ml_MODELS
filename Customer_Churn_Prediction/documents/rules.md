# Development Rules

## Coding Standards

- Follow PEP 8 guidelines.
- Use descriptive variable names.
- Write modular code.
- Keep notebook phases separate.

---

## Dataset Rules

- Never modify the original dataset.
- Save processed data separately.
- Handle missing values before training.
- Remove duplicates.

---

## Machine Learning Rules

- Always use train-test split.
- Compare multiple algorithms.
- Evaluate using multiple metrics.
- Save the trained model.

---

## Visualization Rules

- Save charts inside reports/figures.
- Use proper chart titles.
- Label all axes.
- Avoid unnecessary plots.

---

## Deployment Rules

- Never retrain model in Streamlit.
- Load saved model only.
- Validate user input.
- Show prediction confidence.

---

## Git Rules

Commit after every phase.
